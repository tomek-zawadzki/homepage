TYDZIEŃ I

Co nowego?

  • Frontend – porównanie do samochodu, odpowiada za karoserię i wnętrze używane przez kierowcę i pasażerów.
    • Odpowiada za wygląd strony lub aplikacji internetowej
    • Odpowiada ze interakcję z użytkownikiem
    • Generowany głównie przy użyciu HTML, CSS i JavaScript
    • Technologie frontendowe w niektórych obszarach przejmują rynek tradycyjnych aplikacji
  • Backend – odpowiada za silnik i wszystkie układy które sprawiają, że samochód jedzie (to co pod maską).
  • Konfiguracja potrzebnych narzędzi: Chrome, VSC, Slack, grupa na FB.
  • Uruchomienie pierwszego projektu.

Praca domowa:

  1. Przejrzenie pliku ‘templated-idustrious’ i „pobawienie się nim”, spróbować zmienić treści na stronie, spróbować zmieniać kod i weryfikować co się dzieje.

TYDZIEŃ II

Warstwy strony internetowej

  • Warstwa 0 – treść,
  • Warstwa 1 – HTML warstwa semantyczna, nadaje sens treści strony. Tutaj wprowadzamy znaczniki,
  • Warstwa 2 – CSS warstwa prezentacji, nadaje wygląd poprzednim warstwą,
  • Warstwa 3 – JavaScript warstwa interakcji, najtrudniejszy element frontendu, wpływa na wszystkie poprzednie warstwy,

CONTENT IS THE KING –pamiętaj, że na stronach internetowych to treść jest najważniesza

HTML - HyperText Markup Language - hipertekstowy język znaczników.
Odpowiada za właściwe opakowanie treści strony. Treść strony czyli: teksty, nagłówki, linki, listy, definicje, obrazki, video, formularze, wszystko co jest nośnikiem informacji. Opakowujemy je w znaczniki,tagi i elementy. HTML nie jest językiem programowania, jest językiem znaczników.

Budowa znacznika:

<znacznik atrybut=”wartość”>treść</znaczik>

CSS – Cascading Style Sheets – kaskadowe arkusze stylów.
Odpowiadają z prezentację strony, wpływają na wygląd elementów HTML na ekranie lub na wydrukach. Składnia CSS to reguły {}

Budowa reguły CSS:

selektor {
   właściwość: wartość;
}

Podłączenie pliku .css do HTML – umieszczamy w sekcji <head>, poniżej widzimy wzór takiego podłączenia:

<link href=”style.css” rel=”stylesheet’ type=”tekst/css”>

podłączenie pliku .css do HTML – umieszczamy w sekcji <head>

Czym jest programowanie?

Zamiana ludzkiego języka na język zrozumiały przez komputer Każdy język programowania ma swoją składnię, gramatykę i ortografię Istnieje ponad 700 języków programowania, w tym szkolenie skupimy się na JavaScript

JavaScript:

  • odpowiada za interakcję,
  • pozwala na programowanie we frontendzie,
  • potrafi wpływać na każdą z warstw,
  • rozszerza możliwości zwykłych stron,
  • aktualnie jeden z najpopularniejszych języków programowania na świecie.

TYDZIEŃ III

1. KANBAN – metoda planowania pracy.

W organizowaniu pracy metodą Kanban pomaga nam aplikacja TRELLO Dzielimy pracę na trzy kolumy:

  1. Do zrobienia / to do - tutaj trafiają wszystkie zadanie do zrobienia.
  2. W trakcie / doing - tutaj przerzucamy zadanie aktualnie wykonywane.UWAGA! robimy tylko jedną rzecz na raz.
  3. Zrobione / done - po skończeniu zadanie trafia do tej kolumny

2. NAZEWNICTWO

Jak wpisywać nazwy(np. selektorów):

  • bez polskich znaków,
  • nie używamy spacji,
  • używać angielskich słów,
  • się jednego schematy (konsekwencja jest istotna).

Konwencje nazewnicta:
Przykład: My super bold header
camelCase – mySuperBoldHeader
kebab-case - my-super-bold-header
sneak_case – my_super_bold_header
PascalCase – MySuperBoldHeader

Jak działa internet i przeglądarka

Protokół który obsługuje strony internetowe i ich zawartość to HTTP (obecnie coraz częściej HTTP/2). Komunikacja przeglądarka - serwer odbywa się na zasadzie request - response.

Odpowiedzi serwera mają różne kody:

  • -200-- wszystko OK
  • -30X-- szukasz w złym miejscu - przekieruję Cię w odpowiednie miejsce
  • -404-- błąd - nie mam tego czego potrzebujesz
  • -50X-- błąd wewnętrzny serwera (frontend nic z tym nie zrobi)

Root (/) jest istotną częścią ścieżek do zasobów - trzeba się nauczyć jak on działa.

JavaScript

JavaScript to frontendowe programowanie dzięki, któremu można wykonać wiele ciekawych i szalonych rzeczy. Implemantuje rzeczy ze specyfikacji ECMAScript. ECMAScript to ustandaryzowany skryptowy język programowania.

  4. Przejdź labirynt.

TYDZIEŃ IV

GIT.

GIT jest to system kontroli wersji. GIT składa się z repozytoriów czyli z naszego projektu, a nasz projekt to wszystkie nasze pliki stworzone lub wykorzystane do danego projektu. I tak jak sama nazwa wskazuje możemy go użyć m.in. do takich rzeczy jak: powrót do poprzedniej wersji swojego pliku, odzyskanie skazowanych danych, do zrobienia zmian i powrotu do stanu przed ich dokonaniem, dzielić się projektem w grupie.
Commit = stan projektu i opis tego co się zmieniło od ostatniego zapisu.
Commitujemy jak dokładamy jakąś widoczną zmianę do projektu. Lepiej commitować częściej niż za mało.
Kolejność działania (terminal) - git add . / git commit -m "opis zmiany" / git push

GitHub.

Github != GIT
GitHub jest jednym z serwerów który pozwala aby GIT działał zdalnie. Jest on programistyczną wizytówką więc warto aby nasze konto na GitHubie było zadbane ładne i "prawdziwe".

Praca domowa:

  1. Załóż konto na githubie.
  2. Wrzuć projekt homepage na githuba.
  3. Zacznij commitować zmiany.
  4. Dostosuj style tak, aby ograniczyć szerokość treści i ją wyśrodkuj.
  5. Stwórz pusty element z wybraną klasą i uzupełnij go za pomocą JS.
